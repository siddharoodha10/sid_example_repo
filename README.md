# HMS Requirements Documents

This folder contains module-wise requirements documents (RDs) used for development, testing, and review.

## Files

- `LOGIN_API_RD.html` - Browser-ready requirements document for Login API and token handling.
- `REGISTRATION_API_RD.html` - Browser-ready requirements document for Registration API and user creation rules.
- `MODULE_RD_TEMPLATE.html` - Browser-ready template for the next module RD.
- `LOGIN_API_RD.md` - Markdown source for Login API RD.
- `MODULE_RD_TEMPLATE.md` - Markdown source template.

## RD Rules

- Keep one RD per module or feature area.
- Include API contract, validation rules, database impact, security rules, and test cases.
- Add open questions instead of hiding unclear requirements.
- Update the RD whenever implementation behavior changes.
